# [PRD] 사내 좌석 배치 최적화 시스템: SpaceManager

본 문서는 사내 조직 개편 및 사무 공간 효율화를 위한 좌석 배치 자동화 및 자율 좌석 선택 시스템인 **SpaceManager**의 요구사항을 정의합니다.

---

## 1. 프로젝트 개요
*   **서비스 이름**: SpaceManager
*   **한 줄 설명**: 조직 구조와 상면 데이터를 기반으로 최적의 팀별 구역을 제안하고, 구성원이 자율적으로 좌석을 선택할 수 있게 돕는 '스마트 오피스 지원 솔루션'
*   **핵심 가치**: 
    1.  **배치 업무 효율화**: 복잡한 조직 구조를 반영한 배치 업무의 자동화/최적화
    2.  **데이터 기반 최적화**: 임원-관련 팀 간의 물리적 거리 최적화 및 이동 동선 최소화
    3.  **자율성 부여**: 팀 구역 내에서 구성원이 직접 선호 좌석을 선택하는 유연함 제공

---

## 2. 타겟 유저 및 사용자 시나리오
### 2.1 타겟 유저
1.  **관리자 (Admin)**: 인사팀, 총무팀, 시설관리 담당자 (좌석 배치 기획 및 실행 주체)
2.  **조직 내부 유저 (User)**: 팀 리더 및 팀원 (자신 혹은 팀의 자리를 확인하고 선택하는 주체)

### 2.2 사용자 시나리오
1.  **공간 등록**: 관리자가 현재 사용 중인 층별(4F, 7F, 11F) 상면 이미지와 총 위석 정보를 등록한다.
2.  **데이터 입력**: 관리자가 조직 구조 및 팀별 인원 수(인사 데이터 등)를 엑셀 혹은 수동으로 입력한다.
3.  **배치 시뮬레이션**: 관리자가 '임원-관련 팀 근접 배치' 및 '이동 최소화' 옵션을 설정하여 최적의 구역 배치를 제안받는다.
4.  **확정 및 공지**: 관리자가 최종 배치도를 확정하면, 전사 직원들에게 알림이 발송된다.
5.  **좌석 선택**: 팀원들은 할당된 팀 구역 내에서 본인이 원하는 좌석을 시스템상에서 예약/확정한다.

---

## 3. 핵심 기능 명세
### 3.1 필수 구현 기능 (Must-Have)
*   **조직 구조 관리**: 본부-담당-팀 등 무한 계층 구조를 수용하는 재귀적 조직도 관리 (임원 등 상위 부서 직속 인원 편성 지원)
*   **상면 데이터 관리**: 층별(4F, 7F, 11F) 좌석 수 및 평면 정보 관리 (확장 가능 구조)
*   **배치 최적화 엔진**: 
    - 임원 자리 중심의 관련 부서 근접 배치 알고리즘
    - 현재 배치 대비 변동 폭 최소화(Relocation Cost 최소화) 알고리즘
*   **데이터 인입**: 엑셀 업로드 및 수동 입력 지원
*   **팀내 자율 좌석 선택**: 구역 내 빈 좌석 선택 및 중복 체크 기능

### 3.2 추후 확장 기능 (Nice-to-Have)
*   **상면 이미지 기반 UI**: 층별 도면 이미지 업로드 후 좌표 매핑 기능
*   **실시간 혼잡도 분석**: 부서별 밀집도 및 유휴 좌석 시각화
*   **다국어 지원**: 글로벌 오피스 확장을 위한 언어 팩 적용

## 4. 기술 스택 제안
*   **Frontend**: React (with TypeScript), Tailwind CSS, Lucide React (아이콘)
    - 라이브러리: `react-konva` 혹은 `D3.js` (층별 좌석 배치 시각화 및 드래그 제어)
*   **Backend**: **Kotlin (Spring Boot)**
    - 엔진: 배치 최적화 엔진 (Greedy 혹은 Genetic Algorithm 기반 최적화 라이브러리 활용)
    - 특징: Java와의 높은 호환성, 더 간결하고 안전한 문법 제공 (JVM 기반)
*   **Database**: PostgreSQL (조직도 및 좌석 맵 관계형 데이터 저장)
*   **Infra**: kt cloud DX-M (Cloud-Native 구성), Docker 컨테이너 기반 배포

---

## 5. 주요 화면 구성 및 User Flow
### 5.1 주요 화면 구성
1.  **관리자 대시보드**: 층별 점유율 현황, 조직 데이터 로드 현황 한눈에 보기
2.  **데이터 관리 화면**: 조직도 트리 뷰, 층별 좌석 수 편집기, 엑셀 업로드 모달
3.  **배치 제안 시뮬레이터**: 알고리즘 적용 전/후 비교, 실시간 드래그 앤 드롭 조정
4.  **직원용 좌석 선택 맵**: 자신의 팀 구역 하이라이트, 현재 선점된 좌석 표시

### 5.2 User Flow
*   **데이터 셋업**: 인사 데이터 로드 → 상면 데이터 확인 → 필수 제약조건(임원 위치 등) 고정
*   **시뮬레이션**: '자동 배치 실행' 클릭 → 제안 결과 검토 → 수동 미세 조정 → 최종 승인
*   **좌석 픽업**: 직원 로그인 → 지도에서 본인 팀 구역 확인 → 클릭으로 좌석 확정

## 6. 상세 기능 명세 및 예외 처리 정책
*   **배치 로직**: 
    - 1순위: 임원석 기준 **설정된 가중치(혹은 거리)** 이내 산하 부서 우선 배치 (임원은 상위 조직 직속으로 자동 인식)
    - 2순위: 동일 본부 산하 팀들은 가급적 동일 층 혹은 인접 구역 배치
    - 3순위: 이전 배치(As-Is)에서 이동 거리가 가장 짧은 위치로 배정
*   **오버플로우 처리 (Split Placement)**:
    - 특정 층의 수용 인원을 초과할 경우, 하위 팀 중 일부를 **인접한 다른 층**으로 분산 배치하되, 상위 조직(담당/본부)과의 물리적 거리 합계를 최소화하는 방향으로 제안
*   **팀 단위 경계 및 선택 로직**:
    - **영역 제한**: 각 팀별로 지도상에 다각형(Polygon) 형태의 할당 영역 정의
    - **자율 선택**: 팀원은 본인 소속 팀의 할당 영역 내에 위치한 좌석만 선택 가능
*   **예외 처리**:
    - **가용 좌석 부족**: 총 가용 좌석보다 조직 인원이 많을 경우 경고 문구 및 배치 중단
    - **상면 이미지 미등록**: 이미지 없이 데이터로만 배치 기능 활성화 (Grid 뷰 전환)
    - **동시성 제어**: 좌석 선택 시 동시에 동일 좌석 클릭 시 먼저 트랜잭션 성공한 사용자 우선 (DB Lock 적용)

## 8. 제약사항 및 보안 요구사항
*   **보안**: 사내 전용 서비스로 사내 망(GSLB 혹은 VPN) 접속 권장, LDAP/SSO 기반 로그인 연동
*   **성능**: 수천 명 단위의 조직도 데이터 로딩 및 시뮬레이션 시 3초 이내 응답 보장
*   **라이선스**: 내부 서비스용이므로 GPL 계열 라이선스 포함 오픈소스 활용 가능하나, 패키지 버전 및 의존성 이력 관리 준수
*   **확장성**: 층(Floor) 데이터는 JSON 구조로 가변적으로 추가/삭제가 가능해야 함 (현재 3개 층 → 향후 전 층 확대 대비)
