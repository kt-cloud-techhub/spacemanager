# 질문 형식 가이드 (워크샵)

> 이 파일은 워크샵 환경에서 upstream `question-format-guide.md`를 오버라이드합니다.

## 규칙 요약

| 질문 유형 | 방법 | .md 파일 생성 |
|---|---|---|
| 객관식 | AskUserQuestion 팝업 | ❌ |
| 주관식 | 채팅 입력 | ❌ |
| 답변 기록 | `audit.md`에 추가 | ✅ |

**질문 `.md` 파일 생성 금지. `[Answer]:` 태그 사용 금지.**

---

## 객관식 → AskUserQuestion 팝업

- 한 번에 최대 4개 질문
- 질문당 2~4개 선택지 (자동으로 "기타" 옵션 포함)
- 질문 텍스트는 한국어로 작성
- header는 12자 이내

```yaml
AskUserQuestion:
  question: "사용자 인증 방식은 어떻게 할까요?"
  header: "인증"
  options:
    - label: "로그인 없음"
      description: "팀 내부용이라 인증 불필요"
    - label: "간단한 인증"
      description: "이름만 입력하는 수준"
    - label: "계정 기반"
      description: "아이디/비밀번호로 로그인"
```

## 주관식 → 채팅 입력

- 한 번에 하나의 질문만
- 사용자가 쉽게 답할 수 있도록 예시 답변 함께 제시

```
이 서비스를 누가, 어떤 상황에서 사용하나요?
예: "팀원 10명이 장애 발생 시 티켓을 등록하고, 팀장이 배정하는 방식"
```

---

## 질문 흐름

1. 해당 단계에서 필요한 질문을 모두 수집합니다.
2. 객관식 질문을 먼저 AskUserQuestion으로 표시합니다 (한 번에 최대 4개).
3. 이후 주관식 질문을 채팅으로 하나씩 질문합니다.
4. 모든 답변을 `audit.md`에 기록합니다.

```markdown
## [단계명] - 질문 응답
**Timestamp**: [ISO timestamp]
**질문과 답변**:
- Q: [질문 내용]
- A: [사용자 답변]
---
```

## 모호성 해결

**모순 발견 시** → AskUserQuestion 팝업:
```yaml
AskUserQuestion:
  question: "'로그인 없음'을 선택하셨는데 담당자 배정도 원하십니다. 담당자는 어떻게 구분할까요?"
  options:
    - label: "이름만 입력"
      description: "텍스트로 이름만 적는 방식"
    - label: "담당자 목록에서 선택"
      description: "미리 등록된 팀원 목록에서 선택"
```

**모호한 답변 시** → 채팅 후속 질문:
```
"최대한 간단하게"라고 하셨는데, 꼭 있어야 하는 기능이 무엇인가요?
예: "티켓 생성과 상태 변경만 있으면 돼"
```
