# 콘텐츠 검증 규칙

> **중요**: 파일에 쓰기 전에 모든 콘텐츠를 검증합니다.

## Mermaid 다이어그램

Mermaid 다이어그램이 포함된 파일 생성 전:

1. 노드 ID: 영문자 + 숫자 + 밑줄만 사용합니다.
2. 레이블의 특수문자를 이스케이프합니다: `"` → `\"`, `'` → `\'`.
3. 플로우차트 연결이 문법적으로 올바른지 검증합니다.
4. 다이어그램 아래에 항상 텍스트 대안을 포함합니다.

검증 실패 시 → 텍스트 기반 표현으로 대체합니다. 워크플로우를 중단하지 않습니다.

## ASCII 다이어그램

1. 모든 줄의 너비가 동일해야 합니다.
2. 허용 문자만 사용합니다: `+` `-` `|` `^` `v` `<` `>` 및 공백.
3. 유니코드 박스 그리기 문자 사용 금지.
4. 공백만 사용 — 탭 금지.

## 일반 체크리스트

파일 작성 전:
- [ ] 임베드된 코드 블록(Mermaid, JSON, YAML)이 문법적으로 유효한지 확인
- [ ] 특수문자가 이스케이프되었는지 확인
- [ ] 마크다운 문법이 올바른지 확인
- [ ] 복잡한 시각적 콘텐츠에 텍스트 대안이 제공되었는지 확인

## 검증 실패 시

1. 실패한 내용을 기록합니다.
2. 텍스트 기반 대안을 사용합니다.
3. 워크플로우를 계속 진행합니다 — 중단하지 않습니다.
4. 파싱 제약으로 인해 단순화된 콘텐츠를 사용했음을 사용자에게 알립니다.
